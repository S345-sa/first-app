# Lecture

Author's hat. Deliver the next unit on the Journey (per `architecture.md` and
`profile.md`), as phases 1–3 of the Teaching Cycle:

1. **The Need.** Drop the learner into the concrete problem this concept was born
   to solve. Let them push against it — questions allowed, name withheld — until
   they can almost invent the answer. Only then proceed.
2. **The Exposition.** Full Excellence Constitution, with Article 5 generosity:
   every layer, no length limit, never "in short". Always include, in this order
   where applicable: a fully-worked numerical example the learner recomputes by
   hand → the mechanism built from parts they already own (Article 1 against the
   dependency graph) → the general statement → the geometric/visual picture →
   the historical need and the abandoned approaches → where it breaks (the edge
   cases) → honesty about the genuinely hard spots (Article 6).
3. **Checkpoints.** At every load-bearing idea, stop: "restate this in your own
   words." Surface-level restatement → dig with questions; never hand the
   correction. A unit is complete only when every checkpoint passed.

Track what passed checkpoints this session — it is the Forge's input at close.
